Splash Screen

Technical Description

The Splash Screen (splash.html) serves as the first point of interaction for users visiting the AquaAction website. It does not use the shared HTML template since it is designed as a standalone full-screen experience, but it strictly follows the agreed colour palette and typography to ensure visual consistency with the rest of the site.

HTML Structure: The page uses a <canvas> element as the animated background layer, positioned fixed behind all other content using z-index: 0. Layered <div> elements sit above the canvas to create the gradient overlay, ripple rings, and grain texture effects. The main content is wrapped in a <main> element with position: fixed and flexbox centering to keep everything perfectly centred on any screen size. A <button> element handles the skip interaction, and <meta http-equiv="refresh" content="4;url=home.html"> provides the automatic 4-second redirect as required by the specification.

CSS Techniques: The page makes extensive use of CSS custom properties (variables) that mirror the global style.css palette — ensuring colour consistency without importing the global file. Key animations include @keyframes ring-expand for the five concentric ripple rings, @keyframes spin-arc for the SVG arc loader rotation, and @keyframes rise for the staggered fade-up reveal of each content block using animation-delay. The skip button uses a CSS sweep hover effect achieved with a ::before pseudo-element that translates from translateX(-102%) to translateX(0) on hover, creating a smooth colour fill animation without JavaScript. backdrop-filter: blur() is used on the SDG pill badge to create a frosted glass appearance.

JavaScript: Three JavaScript functions handle the interactive behaviour. The canvas animation loop uses requestAnimationFrame to continuously draw 120 floating particle objects and three layered sine waves at the bottom of the screen, simulating a water environment. A custom cursor is implemented by tracking mousemove events and updating the left and top CSS properties of two fixed <div> elements. The countdown timer uses setInterval to decrement a variable every 1000ms and update the number displayed inside the SVG arc loader. The skipIntro() function calls clearInterval() to cancel the timer and immediately redirects the user using window.location.href.

Accessibility

All decorative elements (canvas, ripple rings, grain overlay, corner watermark) include aria-hidden="true" to prevent screen readers from announcing them. The <main> element uses a descriptive aria-label. The countdown area uses aria-live="polite" so screen readers announce number changes without interrupting the user. The skip button includes a descriptive aria-label attribute. The logo image includes meaningful alt text. The team member section uses aria-label="Team members" on its container. The SVG arc loader uses role="progressbar" with aria-valuenow, aria-valuemin, and aria-valuemax attributes.

Action Impact Simulator

Technical Description

The Action Impact Simulator (ais.html) is a fully interactive JavaScript-driven page that allows users to select water-related actions and receive dynamic feedback on their combined environmental impact. It uses the shared HTML template including the header, nav, and footer, and links to the global style.css with additional page-specific embedded CSS in a <style> block.

HTML Structure: The page uses semantic HTML5 elements throughout. The simulator intro is wrapped in a <section> with aria-labelledby. The six action cards are <article> elements inside a CSS Grid container (.action-grid). Each card uses data-points and data-id custom data attributes to store impact values that JavaScript reads at runtime. The feedback panel is a separate <section> with aria-live="polite" so screen readers announce score updates automatically. A progress bar uses the role="progressbar" ARIA role with live aria-valuenow updates.

CSS Techniques: The action card grid uses grid-template-columns: repeat(auto-fit, minmax(280px, 1fr)) for a fully responsive layout that adjusts columns automatically. Selected cards use a distinct visual state — a teal gradient background, coloured border, and a CSS ::after pseudo-element tick badge — all toggled by adding/removing the .selected class via JavaScript. The progress bar fill uses a CSS linear-gradient and transition: width 0.7s ease for a smooth animated fill effect. The page background overlay uses transition: opacity 1s ease to smoothly cross-fade between the three impact level background images.

JavaScript: A Set object (selectedCards) tracks which card IDs are currently selected, ensuring no duplicate entries. The calcScore() function iterates over the Set and reads each card's data-points attribute to compute the total. The getLevel() function applies defined thresholds (Low: 1–5pts, Medium: 6–12pts, High: 13+pts) to return the current impact level. The updateFeedback() function updates the score number, level label, progress bar width, message text, and background image simultaneously. Event listeners are attached to all cards for both click and keydown events (Enter/Space) to support keyboard interaction. The resetAll() function clears the Set, removes all .selected classes, and resets the feedback panel.

Accessibility

Each action card uses role="checkbox" and aria-checked="true/false" which updates dynamically via JavaScript when a card is selected or deselected, providing full screen reader support for the selection state. Cards include tabindex="0" so keyboard users can Tab to each card and activate it with Enter or Space. All images include descriptive alt text. The feedback panel uses aria-live="polite" and the progress bar uses role="progressbar" with live ARIA value attributes. The active navigation link uses a distinct visual class (.nav-ais) so the current page is clearly indicated.

Sanitation Systems Content Page

Technical Description

The Sanitation Systems content page (content_ST2.html) is Student 2's individual content page exploring SDG 6 Target 6.2 — safe sanitation for all. It uses the shared HTML template with header, navigation bar, main content area, and footer, and links to the global style.css. Page-specific styles are defined in an embedded <style> block and do not override global styles.

HTML Structure: The page is divided into six clearly defined <section> elements, each with a unique id attribute used as anchor targets for internal navigation. Headings follow a strict hierarchy — one <h1> in the hero banner, <h2> for each section, and <h3> for subsections. An internal navigation menu near the top uses an <ol> list of anchor links (href="#section-id") allowing users to jump directly to any section. scroll-margin-top: 90px is applied to each section to prevent content hiding beneath the sticky header when navigating. A fixed "Go to Top" button is implemented using position: fixed in CSS with an anchor link to href="#top" — no JavaScript required.

CSS Techniques: A two-column split layout for image-and-text pairs uses display: grid; grid-template-columns: 1fr 1fr with a @media (max-width: 720px) breakpoint that collapses it to a single column on smaller screens. The stat cards section uses repeat(auto-fit, minmax(180px, 1fr)) to create a responsive four-column row. Image wrappers use overflow: hidden and a CSS transform: scale(1.03) on hover to create a subtle zoom effect. The content hero banner uses a layered background shorthand combining a linear-gradient overlay with a url() background image for text readability over the photo. The callout boxes use a border-left: 5px solid design pattern combined with a soft gradient background to visually distinguish key facts from body text.

Images used in the Content Page:

  • Section 2 — Global Crisis: UN News photo showing lack of water access (news.un.org)
  • Section 3 — Sanitation Systems diagram: ResearchGate sanitation ladder figure (researchgate.net)
  • Section 1 & Hero — Modern sanitation system: JC France Industrie (jcfranceindustrie.fr)
  • Section 5 — Student action: Brainware University World Water Day outreach event (brainwareuniversity.ac.in)

Accessibility

The internal navigation menu uses aria-label="Page sections" to distinguish it from the main navigation. Each content section uses aria-labelledby pointing to its heading ID. All four images include descriptive alt text that explains the content of the image for screen reader users rather than simply naming the file. The stat cards section uses role="region" with an aria-label. The fixed Go to Top button includes a descriptive aria-label and title attribute. The hero banner uses role="banner". Colour contrast between body text and backgrounds meets WCAG AA standards throughout, using the dark abyss colour (#1d5d6f) on light backgrounds.

Challenges and Lessons Learned

Challenge 1 — Canvas animation performance: The initial canvas particle animation caused frame drops on lower-powered devices because the particle pool was too large. This was resolved by reducing the particle count to 120 and optimising the draw loop to avoid unnecessary recalculations inside the animation frame. This taught me the importance of performance-aware JavaScript when using requestAnimationFrame.

Challenge 2 — Meta refresh and JavaScript timer sync: The <meta http-equiv="refresh"> tag and the JavaScript countdown timer needed to stay in sync. Early testing showed the meta refresh would fire slightly before the countdown reached zero due to browser timing differences. The solution was to set the meta refresh to exactly 4 seconds while the JavaScript timer also starts from 4, accepting that a very small discrepancy is unavoidable and does not affect user experience.

Challenge 3 — AIS card selection on mobile: Touch events on mobile devices did not always register on the action cards because of the CSS transition on transform. Adding touch-action: manipulation to the card styles resolved the delay on iOS and Android browsers.

Challenge 4 — Internal navigation hiding under sticky header: On the content page, clicking internal nav links caused sections to scroll into position but the heading would be hidden behind the sticky header. This was resolved by adding scroll-margin-top: 90px to each section, which offsets the scroll position correctly without requiring JavaScript scroll handling.

Lessons Learned: Working on this project developed my understanding of the relationship between HTML structure, CSS layout, and JavaScript interaction. I learned that accessibility is not an afterthought but must be designed in from the start — using semantic HTML and ARIA attributes together. I also learned the value of a shared CSS file for maintaining visual consistency across a multi-page website built collaboratively.
