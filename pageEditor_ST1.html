Home Page

Technical Description

The Home Page (home.html) was designed to introduce users to the mission of AquaAction while maintaining a professional and engaging aesthetic. The page utilizes a semantic HTML5 structure with a <header>, <main>, and <footer>. The Hero section uses a linear-gradient overlay (135deg) on top of a background-image to ensure that the white text remains legible while providing high-impact visuals. Text-shadowing (1px 1px 2px) further enhances clarity against the atmospheric background.

For the "Why we act for SDG 6" section, a .home-card-grid uses CSS Grid with repeat(auto-fit, minmax(300px, 1fr)) to provide a fully responsive layout that automatically adjusts card columns based on screen size. The "Explore our work" gallery uses an .interactive-gallery grid, where each .gallery-item utilizes custom CSS variables (--tile-image) for modular background control. Hover states use transform: translateY(-10px) and a increased z-index: 10 to elevate the active tile without visual clipping.

Accessibility

Accessibility is addressed through the use of semantic HTML5 landmarks and descriptive labels. The primary <nav> includes an aria-label="Main navigation". All images in the content cards have descriptive alt text to assist screen reader users. Heading levels are strictly hierarchical, with <h1> serving as the primary title and <h2> used for subsequent sections. High color contrast is maintained by using the --white text on the dark --dark-abyss background and teal gradients.

Gallery Page

Technical Description

The Gallery Page (gallery.html) showcases SDG 6 innovations through an interactive grid. The layout uses a .gallery-grid container to house six <article> cards. Each card triggers a JavaScript function (openModal(index)) when clicked to display a larger version of the image and its detailed description.

The interactive modal pop-up consists of a .modal-overlay that covers the entire screen, using background: rgba(5, 85, 105, 0.726) and backdrop-filter: blur(8px) to focus the user's attention on the content. The modal's data (title, description, and image source) is dynamically populated from a JavaScript array to ensure efficient content management. A floating action button (.text-size-fab) allows users to toggle a global text size setting via toggleGlobalTextSize(), improving readability for all users.

Accessibility

The gallery uses ARIA attributes to enhance the interactive elements. The modal includes a close button with a clear "x" symbol and an aria-label can be added for further clarity. The thumbnail images use appropriate alt text such as "Rainwater" or "Smart Irrigation". The accessibility FAB includes a title="Toggle Text Size" attribute and an SVG icon to visually indicate its purpose. The headings follow a logical structure from <h1> to <h2>.

Water Conservation Content Page

Technical Description

The Water Conservation page (content_ST1.html) provides educational content about water security. It utilizes an .internal-nav menu with an ordered list of anchor links (href="#id") to allow users to jump to specific sections of the page. The .content-hero banner creates a professional introduction with a min-height: 320px and a gradient overlay for text readability.

Layouts are balanced using a .split-row grid for two-column image and text descriptions, which collapses into a single column on mobile via a @media (max-width: 720px) query. Global statistics are displayed in a .stat-row using repeat(auto-fit, minmax(180px, 1fr)) for responsiveness. A "Go to Top" button (.go-top) is implemented with position: fixed and CSS transitions for a smooth hover effect, ensuring navigation is easy on long pages.

Accessibility

Semantic landmarks like role="banner" for the hero and aria-label="Page sections" for the internal navigation improve screen reader compatibility. Each content section is wrapped in an <article> with an id and aria-labelledby to correctly associate headings with their content. All images include valid alt text, and the "Go to Top" button has an aria-label and title to describe its action.

Challenges and Lessons Learned

Challenge 1 — Template Consistency: One of the biggest hurdles was designing a shared HTML template that would be flexible enough for four different students to use. I had to ensure the style.css variables worked across all layouts without breaking individual page designs. This taught me the value of :root variables for maintaining brand consistency.

Challenge 2 — Gallery Z-Index Bugs: During the Home page development, hovering over a gallery tile sometimes caused it to disappear behind the adjacent tile. This was resolved by explicitly setting z-index: 10 on the :hover state, ensuring the active element always stays on top. This reinforced my understanding of stacking contexts.

Challenge 3 — Modal State Management: In the Gallery page, managing the "hidden" and "visible" states of the modal pop-up while keeping the HTML valid required careful integration of CSS classes and JavaScript. I learned that toggling a class like .open is much more efficient and accessible than manipulating inline styles directly.

Challenge 4 — Internal Navigation Offsets: On the Content page, clicking internal anchor links initially caused the section headings to be hidden under the sticky header. I resolved this by applying scroll-margin-top: 90px to the .content-section class. This was a valuable lesson in modern CSS properties for single-page navigation.
