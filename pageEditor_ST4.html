User Profile Page

Technical Description

The User Profile page (profile.html) was implemented as a progressive, multi-step interface that allows users to build a personal SDG 06 profile. The page structure relies on semantic HTML5 <section> elements and uses a .hidden-section class in CSS to manage the visibility of the profile builder's output cards.

JavaScript and DOM Manipulation: The core functionality is driven by a JavaScript system that uses prompt-based prompts to collect user data. For each of the three steps (Basics, Interests, Actions), custom functions validate input and progressively populate <dl> lists with user responses. The profile completion is visually tracked using a dynamic progress bar; the .progress-bar-fill width is updated via JavaScript to reflect the percentage of steps completed (0%, 33%, 66%, or 100%). Once all steps are finalized, a confirmation message is revealed at the bottom of the card.

Accessibility

The profile builder utilizes aria-labelledby to link sections to their descriptive headings. The progress bar container includes aria-hidden="true" as the progress is already announced by a live text description (e.g., "0% completed") for screen reader users. Interactive buttons are clearly labeled, and the progressive reveal ensures that users with assistive technologies are not overwhelmed by empty containers.

Sitemap Page

Technical Description

The Sitemap page (sitemap.html) features an interactive visual representation of the AquaAction website hierarchy using an inline SVG element. The SVG uses a viewBox="0 0 1300 900" attribute to ensure the diagram remains responsive and scales proportionately across different viewport sizes.

SVG and CSS Interactivity: The hierarchy is hand-coded using <line> elements for branches and <rect> elements for page nodes. Each node is wrapped in an <a> tag to provide direct navigation to the respective page. CSS transitions are applied to the .node-box fill and stroke-width, creating a smooth hover effect when a user interacts with a link. Branch connections are clearly visualized through .branch-line paths that connect the primary navigation trunk to individual content and support pages.

Accessibility

Each node link within the SVG includes a <title> element to provide an accessible name for screen readers. For keyboard accessibility, the .sitemap-link:focus .node-box CSS selector applies a high-visibility orange stroke (#ff9900) with increased width to clearly indicate focus as a user tabs through the diagram. The overall SVG is labeled with an aria-label="Interactive Sitemap".

Future Innovation Content Page

Technical Description

The Content Page (content_ST4.html) explores technological advancements in water security. It follows the group's content structure, featuring a .content-hero banner with a linear-gradient overlay for high text readability.

Layout and Navigation: An internal navigation menu (.internal-nav) allows users to jump to specific sections such as "Smart Water Grids" or "Precision Agriculture" using anchor links. To handle the sticky header, scroll-margin-top: 90px is applied to each content section, ensuring headings are not obscured upon jumping. The "Go to Top" button is implemented with position: fixed in CSS and scroll-behavior: smooth at the HTML level for a seamless user experience. The layout uses a .split-row grid to align imagery with descriptive text, collapsing into a single column for mobile devices.

Accessibility

The page adheres to a strict heading hierarchy, beginning with <h1> in the hero and moving sequentially through <h2> and <h3>. The internal nav is identified with aria-label="Page sections". All images, such as those depicting IoT sensors and agriculture, include descriptive alt text. The stat cards row is designated as a role="region" with an appropriate label for assistive technology.

Challenges and Lessons Learned

Challenge 1 — Progressive Logic: Implementing the JavaScript for the profile builder required careful state management to ensure that steps were unlocked in the correct order and that the progress bar correctly reflected the user's journey. This taught me the importance of clear logic flow when using DOM manipulation.

Challenge 2 — SVG Coordinate Systems: Hand-coding the visual sitemap in SVG was challenging, particularly positioning the nodes and branch lines correctly within the 1300x900 coordinate space while maintaining responsiveness. I learned how to effectively use the viewBox property to solve this.

Challenge 3 — Sticky Header Overlaps: On the content page, internal anchor links initially scrolled the headings behind the sticky header. I resolved this by applying scroll-margin-top to the section targets, which is a modern CSS solution I hadn't used before.

Challenge 4 — Keyboard Focus in SVG: Ensuring the SVG sitemap was fully keyboard accessible required specific focus styling. I learned that default browser outlines don't always apply well to SVG elements, necessitating custom focus states for the node boxes.

Compliance

All pages implemented for Student 4 comply with JANET (Joint Academic NETwork) regulations regarding the publication of web content on university systems. The following measures ensure ethical and responsible use:

Appropriate Content: All information regarding the progressive user profile, sitemap, and water innovations is factual, educational, and strictly relevant to SDG 06. No content encourages illegal behavior or contains offensive material.

Intellectual Property: No proprietary code or copyrighted templates were used. All HTML, CSS, and JavaScript was authored originally. External images were sourced from royalty-free or educational platforms and are correctly credited in the References section.

Responsible Resource Use: External resources like Google Fonts were used responsibly under open licenses. Bandwidth-intensive elements like high-resolution images were optimized to ensure the site remains efficient and lightweight for the university network.

Accessibility Compliance: The implementation follows WCAG 2.1 guidelines to ensure inclusivity, featuring semantic structure, ARIA landmarks, and keyboard-accessible interactive components.
